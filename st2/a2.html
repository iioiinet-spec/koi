void build(int node, int start, int end)

세그먼트 트리의 가장 기본이 되는 단계입니다. 리프 노드에는 배열의 실제 값을 저장하고, 부모 노드에는 두 자식 중 더 큰 값을 저장하며 위로 올라옵니다.

LOGIC
seg[node] = max(seg[node*2], seg[node*2+1]): 부모는 항상 자식들의 최댓값을 보유하게 됩니다.

void push(int node, int start, int end)

Lazy Propagation의 핵심입니다. 미뤄두었던 업데이트 값을 자식 노드로 전파하는 역할을 수행합니다.

WHY?
업데이트 때마다 모든 리프를 방문하면 시간 초과가 납니다. 따라서 "나중에 방문할 때 한꺼번에 처리"하기 위해 이 함수가 필요합니다.[cite: 7]

if (lazy[node] != 0) { // 미뤄둔 일이 있다면
    if (start != end) { // 자식이 있다면 전파
        lazy[node*2] += lazy[node];
        seg[node*2] += lazy[node];
        lazy[node*2+1] += lazy[node];
        seg[node*2+1] += lazy[node];
    }
    lazy[node] = 0; // 전파 완료 후 비우기[cite: 7]
}

void update(int node, int start, int end, int left, int right, long long val)

특정 구간에 물을 더하는 함수입니다. 구간이 완전히 겹치면 더 이상 내려가지 않고 lazy 값만 기록한 뒤 반환하는 것이 포인트입니다.[cite: 7]

TIP
구간 전체에 val을 더하면 그 구간의 **최댓값 역시 val만큼 증가**합니다. 그래서 seg[node] += val을 수행합니다.[cite: 7]

long long query(int node, int start, int end, int left, int right)

구간 내의 최댓값을 찾아 반환합니다. 겹치지 않는 구간을 만났을 때 INF_NEG를 반환하여 최댓값 비교에서 배제되도록 합니다.[cite: 7]


// 겹치지 않는 경우: 최댓값에 영향이 없는 매우 작은 값 리턴[cite: 7]
if (right < start || end < left) return -4000000000000000000LL;

5. 시간 복잡도 상세 분석 (Complexity)

단순 배열 처리와 세그먼트 트리의 성능 차이를 비교해 보세요.[cite: 7]
