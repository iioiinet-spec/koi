문제 : 비버 농장의 물길 관리

지역 성명

1. 문제 설명

비버 마을에는 길게 이어진 N개의 농장이 있다. 농장은 왼쪽부터 오른쪽까지 1번부터 N번까지 번호가 붙어 있다.[cite: 6, 7] 각 농장에는 처음에 정해진 양의 물 Ai가 저장되어 있으며, 관리자는 다음 두 가지 작업을 수행한다.[cite: 6, 7]

  • 작업 1 (1 l r x): l번 농장부터 r번 농장까지 물을 각각 x만큼 더 공급한다.[cite: 6, 7]
  • 작업 2 (2 l r): l번 농장부터 r번 농장까지 중 물이 가장 많은 농장의 물의 양을 출력한다.[cite: 6, 7]

[제약 사항 및 조건]

• N, Q ≤ 100,000 (농장 수 및 명령 수)[cite: 6, 7]
• Ai, x ≤ 100,000 (초기 물의 양 및 증가량)[cite: 6, 7]
• 데이터 범위가 크므로 반드시 long long 타입을 사용해야 한다.[cite: 6, 7]
• 시간 제한: 2.0초 / 메모리 제한: 256MB

[예제 입력]
5 6
3 1 4 2 5
2 1 5
1 2 4 3
2 1 5
2 2 3
1 5 5 10
2 4 5
[예제 출력]
5
7
7
15
심화 강의 및 코드 해설 확인

1. 시간 복잡도 분석

이 문제의 핵심은 N과 Q가 모두 100,000이라는 점입니다.[cite: 6, 7] 단순 반복문으로 구간 업데이트를 수행하면 최악의 경우 O(N*Q)가 되어 약 100억 번의 연산이 필요합니다.[cite: 6, 7] 따라서 O(Q log N)으로 해결 가능한 Lazy Propagation 세그먼트 트리가 필수적입니다.[cite: 6, 7]

2. 핵심 알고리즘: Lazy Propagation

구간 업데이트 시, 하위 노드들을 즉시 갱신하지 않고 lazy 배열에 변경 사항을 기록해 두었다가, 해당 노드를 실제로 방문할 때 갱신(Push)하는 기법입니다.[cite: 6, 7]

• 점 수정: O(log N)
• 구간 수정(Lazy): O(log N)
• 구간 최댓값 질의: O(log N)

3. 정답 소스 코드 (C 언어)

아래 코드는 예제 입력을 완벽하게 통과하며, 실제 대회 환경에서 바로 사용 가능한 표준 코드입니다.


#include <stdio.h>

#define MAXN 100005
#define INF_NEG -4000000000000000000LL // 최댓값 계산을 위한 작은 값[cite: 7]

long long arr[MAXN], seg[MAXN * 4], lazy[MAXN * 4];

long long max(long long a, long long b) { return (a > b) ? a : b; }

// 트리의 기초를 형성하는 build 함수[cite: 7]
void build(int node, int start, int end) {
    if (start == end) {
        seg[node] = arr[start];
        return;
    }
    int mid = (start + end) / 2;
    build(node * 2, start, mid);
    build(node * 2 + 1, mid + 1, end);
    seg[node] = max(seg[node * 2], seg[node * 2 + 1]);
}

// 미뤄둔 업데이트를 자식에게 전달하는 push 함수[cite: 7]
void push(int node, int start, int end) {
    if (lazy[node] != 0) {
        if (start != end) {
            lazy[node * 2] += lazy[node];
            seg[node * 2] += lazy[node];
            lazy[node * 2 + 1] += lazy[node];
            seg[node * 2 + 1] += lazy[node];
        }
        lazy[node] = 0;
    }
}

// 구간에 값을 더하는 update 함수[cite: 7]
void update(int node, int start, int end, int left, int right, long long val) {
    push(node, start, end);
    if (right < start || end < left) return;
    if (left <= start && end <= right) {
        seg[node] += val;
        lazy[node] += val;
        return;
    }
    int mid = (start + end) / 2;
    update(node * 2, start, mid, left, right, val);
    update(node * 2 + 1, mid + 1, end, left, right, val);
    seg[node] = max(seg[node * 2], seg[node * 2 + 1]);
}

// 구간 최댓값을 구하는 query 함수[cite: 7]
long long query(int node, int start, int end, int left, int right) {
    push(node, start, end);
    if (right < start || end < left) return INF_NEG;
    if (left <= start && end <= right) return seg[node];
    int mid = (start + end) / 2;
    return max(query(node * 2, start, mid, left, right), 
               query(node * 2 + 1, mid + 1, end, left, right));
}

int main() {
    int N, Q;
    scanf("%d %d", &N, &Q);
    for (int i = 1; i <= N; i++) scanf("%lld", &arr[i]);
    
    build(1, 1, N);
    
    while (Q--) {
        int type, l, r;
        scanf("%d %d %d", &type, &l, &r);
        if (type == 1) {
            long long x; scanf("%lld", &x);
            update(1, 1, N, l, r, x);
        } else {
            printf("%lld\n", query(1, 1, N, l, r));
        }
    }
    return 0;
}

🎓 강의 마무리 및 핵심 요약

  • 구간 업데이트가 포함된 문제는 100% Lazy Propagation 문제입니다.[cite: 6, 7]
  • 최댓값을 관리할 때는 구간 합과 달리 노드 범위 길이를 곱하지 않고 단순히 더해줍니다.[cite: 7]
  • push 함수는 자식 노드로 내려가기 직전(update, query 시작부)에 반드시 호출해야 합니다.[cite: 7]
