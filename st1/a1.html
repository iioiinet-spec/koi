문제 : 비버 마을의 에너지 나무

비버 마을에는 1번 마을회관을 중심으로 여러 집이 나무(Tree) 구조로 연결되어 있다. 마을에는 총 N개의 집이 있고, 집들은 N-1개의 길로 연결되어 있다. 모든 집은 길을 따라 1번 집까지 갈 수 있으며, 길 사이에 순환은 존재하지 않는다.

각 집에는 에너지 저장 장치가 있으며, 초기 에너지 양 Ai가 주어진다. 마을에서는 다음과 같은 두 종류의 작업이 빈번하게 발생한다.

  • 1 u x: u번 집의 에너지를 x로 변경한다.
  • 2 u: u번 집을 중심으로 하는 서브트리 전체의 에너지 합을 구한다.

단순 DFS 방식은 시간 초과가 발생하므로, 트리를 선형 배열로 변환한 뒤 세그먼트 트리 등을 사용하여 효율적으로 해결해야 한다.[cite: 5]

[제약 조건]

• N, Q (1 ~ 100,000)
• Ai, x (1 ~ 100,000)
• 결과값은 32비트 정수 범위를 넘을 수 있으므로 long long을 사용한다.

[예제 입출력]

Input
7 5
5 3 8 2 4 6 1
1 2
1 3
2 4
2 5
3 6
3 7
2 1
2 2
1 5 10
2 2
2 3
Output
29
9
15
15
해설 및 강의 모드 전환

1. 문제 해결 전략 (Strategy)

이 문제의 핵심은 "트리 구조에서의 구간 쿼리"입니다.[cite: 5] N과 Q가 각각 10만인 경우, 단순 DFS로 매번 합을 구하면 O(N*Q), 즉 100억 번의 연산이 필요하여 시간 초과가 발생합니다.[cite: 5]

최적화 방법:
트리를 DFS 순서로 배열에 펼치는 오일러 투어 테크닉을 적용하여 O((N+Q) log N)으로 성능을 개선해야 합니다.

2. 오일러 투어 테크닉 (ETT)

트리의 서브트리 구조를 배열의 연속된 구간으로 변환하는 마법 같은 기법입니다.

1
진입/탈출 시간 기록: DFS를 돌며 노드에 들어갈 때(in)와 나올 때(out)의 타이머를 기록합니다.
2
구간의 발견: u번 노드의 서브트리는 배열의 [in[u], out[u]] 구간에 완벽하게 대응됩니다.
3
차원 축소: 이제 트리 문제는 "배열의 특정 구간 합"을 구하는 문제로 바뀝니다.

3. 전체 소스 코드 해설

실제 대회에서 사용 가능한 효율적인 C 코드입니다.

#include <stdio.h>

#define MAXN 100005

int N, Q, timer;
int head[MAXN], to[MAXN * 2], nextEdge[MAXN * 2], edgeCount;
int in[MAXN], out[MAXN];
long long value[MAXN], arr[MAXN], seg[MAXN * 4];

// 트리의 인접 리스트 생성[cite: 5]
void addEdge(int u, int v) {
    to[edgeCount] = v;
    nextEdge[edgeCount] = head[u];
    head[u] = edgeCount++;
}

// 오일러 투어: 서브트리를 배열 구간으로 매핑
void dfs(int now, int parent) {
    in[now] = ++timer;
    for (int e = head[now]; e != -1; e = nextEdge[e]) {
        if (to[e] != parent) dfs(to[e], now);
    }
    out[now] = timer;
}

// 세그먼트 트리: 초기 구축
void build(int node, int start, int end) {
    if (start == end) {
        seg[node] = arr[start];
        return;
    }
    int mid = (start + end) / 2;
    build(node * 2, start, mid);
    build(node * 2 + 1, mid + 1, end);
    seg[node] = seg[node * 2] + seg[node * 2 + 1];
}

// 세그먼트 트리: 특정 위치 값 수정[cite: 5]
void update(int node, int start, int end, int idx, long long val) {
    if (start == end) {
        seg[node] = val;
        return;
    }
    int mid = (start + end) / 2;
    if (idx <= mid) update(node * 2, start, mid, idx, val);
    else update(node * 2 + 1, mid + 1, end, idx, val);
    seg[node] = seg[node * 2] + seg[node * 2 + 1];
}

// 세그먼트 트리: 구간 합 쿼리[cite: 5]
long long query(int node, int start, int end, int l, int r) {
    if (r < start || end < l) return 0;
    if (l <= start && end <= r) return seg[node];
    int mid = (start + end) / 2;
    return query(node * 2, start, mid, l, r) + query(node * 2 + 1, mid + 1, end, l, r);
}

int main() {
    scanf("%d %d", &N, &Q);
    for (int i = 1; i <= N; i++) head[i] = -1;
    for (int i = 1; i <= N; i++) scanf("%lld", &value[i]);
    for (int i = 0; i < N - 1; i++) {
        int u, v;
        scanf("%d %d", &u, &v);
        addEdge(u, v); addEdge(v, u);
    }

    // 트리 전처리 및 일렬화
    dfs(1, 0);
    for (int i = 1; i <= N; i++) arr[in[i]] = value[i];
    build(1, 1, N);

    // 쿼리 처리[cite: 5]
    while (Q--) {
        int type, u;
        scanf("%d %d", &type, &u);
        if (type == 1) {
            long long x;
            scanf("%lld", &x);
            update(1, 1, N, in[u], x);
        } else {
            printf("%lld\n", query(1, 1, N, in[u], out[u]));
        }
    }
    return 0;
}

실전 팁 (Cheat Sheet)

  • 트리 문제를 만났는데 "특정 노드 아래의 모든 값"을 묻는다면 오일러 투어를 의심하세요.[cite: 5]
  • 값의 업데이트가 빈번할 때는 세그먼트 트리가 1순위 선택지입니다.[cite: 5]
  • 메모리가 타이트하다면 세그먼트 트리 대신 펜윅 트리(Fenwick Tree)를 고려해 보세요.
